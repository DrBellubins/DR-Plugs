================================================================================
INDUSTRY STANDARD UI TERMINOLOGY
================================================================================

This document maps the elements described in the ModuleContainer design to their
industry-standard names across audio plugin development, general UI/UX, and
hardware design.

================================================================================
CONTAINER-LEVEL ELEMENTS
================================================================================

YOUR TERM                → INDUSTRY STANDARD NAMES
─────────────────────────────────────────────────────────────────────────────────
Module Container         → "Effects Rack", "Module Rack", "Panel Strip",
                           "Card Container", "Horizontal Module Bay"
                           (RC-20 calls it: "Effects Section")

Container Background     → "Canvas", "Backdrop", "Surface" (Material Design)

Container Outline        → "Border", "Stroke", "Frame", "Bezel" (hardware)

================================================================================
MODULE-LEVEL ELEMENTS
================================================================================

YOUR TERM                → INDUSTRY STANDARD NAMES
─────────────────────────────────────────────────────────────────────────────────
Module                   → "Module", "Effects Module", "FX Unit", "Card",
                           "Panel", "Section", "Channel" (modular synth: "Module")
                           (RC-20 calls each one: "Effect Module")

Module Background        → "Module Surface", "Card Background", "Panel Fill"

Module Theme Color       → "Accent Color", "Brand Color", "Module Tint",
                           "Hue Identity"

Enable/Disable Button    → "Bypass Toggle", "Power Switch", "Enable Switch",
                           "Active Indicator", "Module Bypass"
                           (Hardware: "Footswitch", "Power LED")

Grayed-out/Disabled      → "Bypassed State", "Inactive State", "Dimmed State",
                           "Muted State" (opacity reduction = "ghosting")

================================================================================
OSCILLOSCOPE / WAVEFORM DISPLAY
================================================================================

YOUR TERM                → INDUSTRY STANDARD NAMES
─────────────────────────────────────────────────────────────────────────────────
Oscilloscope             → "Oscilloscope", "Scope", "Waveform Display",
                           "Waveform Monitor", "Signal Display"

Animated Waveform        → "Real-time Scope", "Live Waveform", "Scrolling Scope",
                           "Time-domain Display"

Fixed-frame Waveform     → "Transfer Function Display", "Waveshaper Curve",
                           "Static Waveform", "Characteristic Curve",
                           "I/O Curve" (input/output mapping)

                           "Center Indicator"

Lookahead                → "Lookahead Buffer", "Preview Window",
                           "Predictive Display"

Lookback                 → "History Buffer", "Trail", "Lookback Window"

                           "Mode Selector", "Picker", "Choice Box"

                           "Toggle Switch" (circular variant),
                           "LED Button" (hardware analogy)

Label                    → "Caption", "Control Label", "Parameter Label",
                           "Descriptor"

Contextual Label         → "Dynamic Label", "Context-sensitive Caption",
                           "Adaptive Label", "Mode-dependent Label"

================================================================================
RC-20 SPECIFIC TERMINOLOGY (for reference)
================================================================================

RC-20 uses these terms for its similar UI:
- The horizontal strip of modules = "Effects Chain"
- Each module panel = "Effect Module" (Noise, Wobble, Distort, Digital, Space, Mag)
- The waveform at top of each module = "Display" / "Scope"
- The controls below = "Parameters"
- The power button per module = "On/Off Switch"
- The overall look = "Vintage Console" aesthetic
