================================================================================
RECOMMENDATION
================================================================================

PRIMARY: Implementation 6 (Hybrid Pre-Write with Blend)
  - Best match to developer's description
  - Produces swell + reverb from single chain
  - 8 allpass operations per sample (3x faster than current code)
  - Blend between clean and diffused path matches developer's exact words

SECONDARY: Implementation 2 (Delay-Offset) if 6 is too complex
  - Simplest possible version that still produces good swell
  - Gain ramp alone (no explicit blend) may be sufficient

FALLBACK: Implementation 4 (Pure Wet/Dry Blend)
  - If the swell effect isn't critical and you just need the blend behavior
  - Most literal reading of "blend between diffused and normal"
  - Easiest to implement but swell quality depends entirely on compensation

================================================================================
KEY ARCHITECTURAL CHANGE FROM CURRENT CODE
================================================================================

CURRENT (your code):
  - 3 separate DiffusionChain pairs (delayWrite, delayRead, reverb)
  - Up to 24 allpass stages processed per sample per channel
  - Complex crossfading logic between chains
  - Separate tunings for delay vs reverb modes

PROPOSED (any of the above):
  - 1 DiffusionChain per channel
  - Exactly 8 allpass stages per sample per channel
  - Single parameter (gain or blend) controls the effect
  - One set of tunings

The performance improvement is approximately 3x fewer allpass operations,
plus elimination of crossfade math and dual-path processing overhead.
